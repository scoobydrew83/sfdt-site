[Semantic Versioning](https://semver.org/). Each workspace maintains its own changelog in the
repository; this page summarizes recent highlights.

## `@sfdt/cli` тАФ 0.18.0

A platform-integrity release plus a new API-version audit across every surface:

- **[`sfdt versions`](/cli/commands/org-health#sfdt-versions)** тАФ audit the Salesforce API
  versions of local source (Apex, Flow, LWC, Aura) and the org against the org's max API version;
  `--local-only` works offline, `--json` for scripting.
- **[`sfdt versions --advise`](/cli/commands/org-health#sfdt-versions)** тАФ an AI upgrade advisor
  (value / risks / breaking changes / effort per component), grounded by a curated per-version
  registry so it never invents version facts; read-only.
- **[Surface catalog framework](/cli/commands)** тАФ the command reference, feature counts, and
  dashboard page list are now generated from the code's own catalogs, with CI that fails on drift.
- **[GitHub Action `args-json`](/cli/github-action)** тАФ a shell-injection-proof way to pass
  arguments; the legacy `command` string is deprecated and hardened.
- **Hardening** тАФ `test --logic` zero-test guard + `--wait` validation, and
  [`quality --allow-legacy-analyzer`](/cli/commands/testing-quality#sfdt-quality) (Code Analyzer
  v4 is opt-in legacy only; v5 required for authoritative scans).

## `@sfdt/extension` тАФ 0.7.1

Quick-win tooling upgrades across the SOQL Runner, log tools, and record inspector тАФ Phase 1 of
the extension build-out. See [Features](/chrome-extension/features).

- **SOQL Runner power-ups** тАФ a **ЁЯФО Explain** action shows Salesforce's query plan
  (cost, cardinality, leading operation, notes) without executing; **Copy JSON** and
  **Copy for Excel (TSV)** on the result toolbar; and **Export all as CSV** that follows
  `queryMore` pagination to download every row, with a live progress indicator and Cancel
  (memory-safe by construction).
- **Query template pack** тАФ the Saved SOQL panel gains a read-only **Templates** group of 8
  built-in admin/dev queries (Apex test coverage, validation rules, login history, setup audit
  trail, limits snapshot, deploy requests, active TraceFlags, record types) that load straight
  into the runner.
- **Execute Anonymous тАФ DebugLevel picker** тАФ choose which org DebugLevel a captured log uses from
  a native **Log level** select (persists per user), no trip to Setup.
- **Debug Logs тАФ auto-refresh + bulk delete** тАФ a 15-second **Auto-refresh** toggle (off by
  default, no orphan timers) and a **Delete all logs** action that clears every org `ApexLog`
  behind a count-confirm dialog.
- **Inspect Record тАФ raw-JSON toggle + right-click open** тАФ a **Fields / JSON** view toggle renders
  the raw REST payload (copyable), and a right-click **"Inspect this record"** context menu opens
  the inspector for a record Id on the page or link (new `contextMenus` permission тАФ see
  [Privacy &amp; Permissions](/chrome-extension/privacy)).
- **Login as user** тАФ the Setup tabs feature adds a **Login as userтАж** deep link to Setup's user
  list (deep link only; Salesforce enforces the Login-As permission server-side).

## `@sfdt/cli` тАФ 0.17.0

A CI/CD and diagnostics batch:

- **[GitHub Action](/cli/github-action)** тАФ run any sfdt command as a single step
  (`uses: scoobydrew83/sfdt@v0`) with built-in sfdx-url/JWT auth and version pinning
  (`cli-version: auto`).
- **[`ci init` expansion](/cli/ci-cd#generating-a-pipeline--sfdt-ci-init)** тАФ new `release`
  (approval-gated real deploys) and `scratch` (create тЖТ deploy тЖТ test тЖТ always-delete) pipeline
  types; JWT auth (`--auth jwt`); Docker (`--runner docker`, GitLab/Bitbucket) and Action
  (`--runner action`, GitHub) runners.
- **Quality gate in PR pipelines** тАФ generated deploy workflows run an advisory `sfdt quality`
  scan; GitHub uploads SARIF to code scanning
  ([`quality --output-file`](/cli/commands/testing-quality#sfdt-quality)).
- **[`test --lwc`](/cli/commands/testing-quality#sfdt-test)** тАФ run LWC (Jest) unit tests via the
  CLI, locally and in scratch-org CI.
- **[`deploy --smart --notify`](/cli/smart-deploy#notifications)** тАФ smart deploys now push
  success/failure notifications through your configured channels.
- **[`sfdt doctor`](/cli/commands/platform-bridge#sfdt-doctor) core group** тАФ new environment
  diagnostics (`sf`, Node, `git`, `.sfdt` config validity, AI provider, and a warn-only,
  timeout-bounded org-connectivity probe) alongside the extension-stack checks. Flags `--core`,
  `--extension`, `--org`, `--json`; CI-safe (the org check never fails the run).
- **Node 22 floor** тАФ @sfdt/cli requires Node тЙе 22.15; all generated pipelines and the action
  default to Node 22.

## `@sfdt/cli` тАФ 0.16.0

A Salesforce-release-readiness and Agentforce batch:

- **[`sfdt agent-test`](/cli/commands/testing-quality#sfdt-agent-test)** тАФ run an Agentforce agent
  test (`sf agent test run`) as an exit-code CI gate, with `--notify` and `--pr-comment`. Smart
  deploy and `sfdt manifest` now also recognise **Agentforce metadata** (Bot, GenAiPlanner/Plugin,
  AiEvaluationDefinition, and more) in git deltas тАФ see [Smart Deploy](/cli/smart-deploy).
- **[`sfdt test --logic`](/cli/commands/testing-quality#sfdt-test)** тАФ run Apex **and** Flow tests
  together via Salesforce's Spring '26 `sf logic run test`, with AI failure analysis.
- **[`sfdt quality`](/cli/commands/testing-quality#sfdt-quality) upgrades** тАФ **Code Analyzer v5**
  (`sf code-analyzer run`) with `--include-fixes`, an API **v67** user-mode readiness scan
  (`--api67`), and a `--test-hints` check for missing `@IsTest(testFor=тАж)` annotations.
- **[`sfdt deploy`](/cli/commands/core#sfdt-deploy) automations as flags** тАФ `--tag`, `--create-pr`,
  and `--notify` (previously GUI-only), plus any-`.xml` manifest auto-detection.
- **New [org-health checks](/cli/commands/org-health#sfdt-audit)** тАФ `audit mfa-readiness` and
  `audit soap-logins`; `monitor limits`/`org-info` gain elastic-async and release-version reporting.
- **[Google Chat notifications](/cli/notifications)** and **[dashboard](/cli/dashboard)** run-now /
  create-delete-generate actions for Audit, Monitor, Scratch, Data, and Docs (plus new **Agent Test**
  and **Retrofit** pages).
- **[MCP surface now covers the full CLI lifecycle](/cli/mcp)** тАФ on top of the read-only
  `sfdt_coverage` / `sfdt_scan` / `sfdt_dependencies` / `sfdt_flow_scan`, the server gained the
  mutating (`confirmExecution`-gated) `sfdt_release`, `sfdt_scratch_*`, and `sfdt_data_*` tools plus
  `sfdt_test` and read-only `sfdt_history` тАФ 30 tools in total.
- **[Run history](/cli/commands/org-health#sfdt-history)** тАФ `sfdt history` (with `--type` / `--limit`
  / `--json`) surfaces a durable local index of every audit/monitor/quality/test/deploy/agent-test run,
  so outcomes can be trended over time; `audit`/`monitor` also archive full timestamped snapshots.
- **[`sfdt test --class-names`](/cli/commands/testing-quality#sfdt-test)** runs a specific subset of
  Apex test classes (powers the VS Code "Run test class" CodeLens), and **[`sfdt skills export
  --target pack`](/cli/commands/platform-bridge)** emits an `npx skills add`-compatible pack. The
  Chrome extension's native-messaging host now also answers the read-only bridge kinds
  (`quality`/`scan`/`compare`/`drift`/`org-health`) when the HTTP bridge isn't running.
- **[Dependency](/cli/commands/metadata#sfdt-dependencies) coverage expanded + graph rebuilt** тАФ the
  CLI and the dashboard graph share one `METADATA_TYPE_REGISTRY` in `@sfdt/flow-core` (no more drift)
  and cover **Visualforce components and Aura** alongside Apex, triggers, VF pages, Flows, and LWC,
  with **Custom Object / Custom Field** reachable too. The [dashboard](/cli/dashboard) graph is now
  **seed + expand-on-click**: pick a component by name and type, then click nodes to expand their
  dependencies (both directions, capped per click with a **"+more"** badge on high-degree nodes) тАФ so
  it scales to any org instead of bulk-loading the whole thing.
- **[Dependency gap report](/cli/commands/metadata#source-parsed-gaps---gaps)** тАФ `sfdt dependencies
  <name> --gaps` parses local source to surface references the Tooling API misses (dynamic Apex, LWC
  `@salesforce/apex`, formula and Flow references). Offline by default; `--org` diffs against the API
  to mark each **MISSING** vs **confirmed**. Also available as a **Gaps** panel on the dashboard
  Dependencies page, and a **Show inferred** toggle overlays those missing edges onto the graph
  itself as dashed lines.

## `@sfdt/cli` тАФ 0.15.2

- **Fewer false-positive trigger conflicts** тАФ `@sfdt/flow-core` 0.9.4 now reads a record-triggered
  flow's event from `recordTriggerType`, so Create-only and Update-only flows no longer collapse into
  a single "Unknown" conflict bucket in `sfdt audit` / Flow Intelligence. The fix reaches every
  flow-core consumer: CLI, dashboard, Chrome extension, and VS Code extension.
- **VS Code dashboard embed fixed** тАФ the dashboard server now sends
  `Content-Security-Policy: frame-ancestors 'self' vscode-webview:` instead of
  `X-Frame-Options: SAMEORIGIN`, so the [VS Code extension](/vscode-extension)'s webview can embed
  the dashboard again (it previously rendered blank). Arbitrary web origins are still refused.

## `@sfdt/cli` тАФ 0.13.1

Adds a native **org health & operations** suite тАФ clean-room reimplementations of org
diagnose/audit, monitoring/backup, documentation generation, data-set management, and scratch-org
pooling тАФ surfaced across four consumers: the CLI, the web dashboard, the built-in
[MCP server](/cli/mcp), and a brand-new VS Code extension. No AGPL dependency.

- **`sfdt audit` / `sfdt monitor`** тАФ native org diagnostics and monitoring (audit trail,
  licenses, MFA coverage, unused Apex, inactive users, deprecated API versions; org limits,
  errors, health, plus metadata backup), each writing a JSON snapshot every surface can read.
- **`sfdt docs` / `sfdt data` / `sfdt scratch`** тАФ generate MkDocs docs (objects, Apex, Flows,
  LWC) with optional AI overviews and per-role Developer/Admin/User/DevOps guides; manage data
  sets via `sf data tree`; and create/delete/list scratch orgs with a pre-created pool.
- **VS Code extension (`@sfdt/vscode`)** тАФ a new fourth surface: command-palette integration, an
  **Org Health** sidebar (audit + monitor snapshots), a status-bar item, and an embedded
  dashboard webview.
- **0.13.1 hardening** тАФ `sfdt data delete` no longer reports partial or failed deletes as clean
  (`--json` returns `status: "partial"` with `skippedCount`/`errorCount`); `audit`, `monitor`,
  and `data` now surface Salesforce CLI's real error text instead of the opaque execa message.
