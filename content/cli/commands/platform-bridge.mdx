# Platform & Bridge

These commands run the web dashboard, manage the Chrome extension bridge, start the MCP server,
and extend sfdt.

---

## `sfdt ui`

Starts the local web dashboard (an Express server bound to `127.0.0.1`) and opens it in your
browser. See the full [Web Dashboard](/cli/dashboard) page.

```bash
sfdt ui                   # opens http://localhost:7654
sfdt ui --port 8080       # custom port
sfdt ui --no-open         # start without opening the browser
```

| Option | Description |
|---|---|
| `-p, --port <n>` | Port to listen on (default `7654`) |
| `--no-open` | Start without opening the browser |

<Callout type="info">
  When installing from source, build the dashboard first with `npm run build:gui`. The published
  npm package ships a pre-built `gui/dist/`.
</Callout>

## `sfdt extension`

Manages the Chrome native messaging host that backs the extension's fallback transport. See
[The Bridge](/guides/the-bridge) and the [Chrome Extension](/chrome-extension/bridge) docs.

```bash
sfdt extension install-host --extension-id abcdefghijklmnopabcdefghijklmnop --browser chrome
sfdt extension install-host --extension-id <id> --browser all
sfdt extension status
sfdt extension uninstall-host --browser all
sfdt extension stats --limit 20
```

| Subcommand | Options |
|---|---|
| `install-host` | `--extension-id <id>` (required, 32 lowercase a–p chars), `--browser <chrome\|edge\|brave\|chromium\|vivaldi\|all>`, `--json` |
| `uninstall-host` | `--browser <name>`, `--json` |
| `status` | `--json` |
| `stats` | `--limit <n>` (default `10`), `--json` |

## `sfdt feature-flags`

Manages `.sfdt/feature-flags.json` to remotely disable specific extension features — a kill
switch that needs no Web Store re-review.

```bash
sfdt feature-flags list
sfdt feature-flags disable flow-health-check
sfdt feature-flags enable flow-health-check
sfdt feature-flags clear --remove
```

| Subcommand | Description |
|---|---|
| `list` | Show the current kill-switch file |
| `disable <id>` | Add a feature id to the kill list |
| `enable <id>` | Remove a feature id |
| `clear` | Re-enable everything (`--remove` deletes the file) |

## `sfdt doctor`

End-to-end local diagnostic. Two check groups run by default:

- **Environment** — `sf`, `node`, and `git` are present; `.sfdt/` config is valid; the
  configured AI provider is reachable (when enabled); and the default org is reachable
  (always run, but warn-only — it never fails the command and is bounded by a timeout).
- **Extension stack** — bridge reachable, native host registered, kill-switch readable,
  telemetry snapshot present.

With no flag, both groups run. Exits non-zero if any check fails (org connectivity never
fails) — wire it into CI.

```bash
sfdt doctor                 # environment + extension groups
sfdt doctor --core          # environment checks only
sfdt doctor --extension     # extension checks only
sfdt doctor --org myAlias   # check a specific org's connectivity
sfdt doctor --json          # CI-friendly structured output
```

| Option | Description |
|---|---|
| `--core` | Run only the environment checks (sf, node, git, config, AI, org) |
| `--extension` | Run only the extension-stack checks |
| `--org <alias>` | Org alias for the connectivity check (default: config `defaultOrg`) |
| `--json` | CI-friendly structured output |
| `--port <n>` | Bridge port if not the default |

## `sfdt mcp`

Starts the built-in Model Context Protocol server so agents can drive sfdt as a tool. See the
full [MCP Server](/cli/mcp) page.

```bash
sfdt mcp start      # JSON-RPC 2.0 over stdio
sfdt mcp cleanup    # purge expired parked results from the cache
```

| Subcommand | Description |
|---|---|
| `start` | Start the MCP server (stdio transport) |
| `cleanup` | Purge expired parked results from `.sfdt/cache/parked/` |

## `sfdt plugin`

Scaffolds a new sfdt CLI plugin project. See [Plugins](/cli/plugins).

```bash
sfdt plugin create my-plugin --description "My custom command" --author "you@example.com"
```

| Option | Description |
|---|---|
| `--description <desc>` | Plugin description |
| `--author <author>` | Plugin author |

## `sfdt skills`

Exports SFDT's bundled agent skills to IDE/agent-specific configurations, or to an
[`npx skills`](https://github.com/vercel-labs/skills)-compatible pack.

```bash
sfdt skills export                              # IDE rules (claude, default)
sfdt skills export --target cursor
sfdt skills export --target pack --out ./sfdt-skills-pack
sfdt skills export --json
```

| Option | Description |
|---|---|
| `--target <t>` | `claude` (default) \| `cursor` \| `codex` \| `windsurf` — IDE/agent rules; or `pack` for an `npx skills add`-compatible pack |
| `--out <dir>` | Output directory for `--target pack` (default `./sfdt-skills-pack`) |
| `--json` | Structured output |

<Callout type="info">
  `--target pack` emits a root `manifest.json` plus a copy of every skill folder, mirroring the
  [vercel-labs/skills](https://github.com/vercel-labs/skills) schema — so coding agents can install
  SFDT's Salesforce workflows the same way as `forcedotcom/sf-skills`. The published npm package
  ships the `skills/` folder, so the export works from a global install.
</Callout>
