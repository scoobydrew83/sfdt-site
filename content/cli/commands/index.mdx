# Command Reference

sfdt ships **35 commands**. They are grouped here by area. Every command accepts `--help`, and
most operational commands accept `--json` for scripting and `--dry-run` to preview.

## Core & Deployment

| Command | Description |
|---|---|
| [`sfdt init`](/cli/commands/core#sfdt-init) | Initialize `.sfdt/` config (interactive) |
| [`sfdt deploy`](/cli/commands/core#sfdt-deploy) | Deploy metadata to a Salesforce org |
| [`sfdt preflight`](/cli/commands/core#sfdt-preflight) | Run pre-deployment validation checks |
| [`sfdt rollback`](/cli/commands/core#sfdt-rollback) | Roll back a deployment (with pre-rollback backup) |
| [`sfdt smoke`](/cli/commands/core#sfdt-smoke) | Post-deploy smoke tests |
| [`sfdt release`](/cli/commands/core#sfdt-release) | Generate a release manifest + optional AI release notes |
| [`sfdt test`](/cli/commands/testing-quality#sfdt-test) | Run Apex tests with the enhanced runner |
| [`sfdt pull`](/cli/commands/metadata#sfdt-pull) | Pull metadata from the configured org (incremental) |

## Metadata & Source Control

| Command | Description |
|---|---|
| [`sfdt manifest`](/cli/commands/metadata#sfdt-manifest) | Build `package.xml` from git diffs |
| [`sfdt drift`](/cli/commands/metadata#sfdt-drift) | Detect metadata drift between local source and an org |
| [`sfdt compare`](/cli/commands/metadata#sfdt-compare) | Compare metadata between two orgs or local vs org |
| [`sfdt scan`](/cli/commands/metadata#sfdt-scan) | Fetch a full metadata inventory from an org |

## Testing & Quality
