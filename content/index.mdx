## Why SFDT

- **Built for Salesforce DX** — no project-specific values are hardcoded. Point it at any repo and run `sfdt init`.
- **Safe by default** — preflight validation, pre-rollback backups, post-deploy smoke tests, and a local-only, token-protected bridge.
- **AI when you want it, never required** — code review, error explanation, PR descriptions, and release notes work with Claude, Gemini, or OpenAI — and degrade gracefully to heuristics when AI is off.
- **One shared engine** — the `@sfdt/flow-core` library and a versioned bridge contract keep the CLI, Chrome extension, and VS Code extension in lockstep.
