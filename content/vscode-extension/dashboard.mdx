# The Embedded Dashboard

The **SFDT: Open Dashboard** command (also the Open Dashboard button in the Org Health view, and
clicking the status-bar item) launches the full [`sfdt ui`](/cli/dashboard) web dashboard
**inside an editor tab** — no separate browser window.

## How it works

1. The extension spawns a `sfdt ui` child process.
2. It polls the server's `/api/health` endpoint (up to ~15 seconds) until the server is ready.
3. It opens a VS Code `WebviewPanel` and loads the dashboard in an iframe pointed at the local
   server.

The panel retains its state when hidden, so the dashboard keeps running until you close the tab.

## Port

The dashboard server port is controlled by [`sfdt.dashboardPort`](/vscode-extension/settings)
(default `7654`). Change it if `7654` is already in use on your machine.

## Theme

The embedded dashboard follows your **VS Code light/dark theme** — the extension passes the editor's
current theme to the dashboard, and the panel updates live when you switch themes. Opening `sfdt ui`
directly in a browser is unaffected: it uses its own theme toggle and remembers your last choice.

<Callout type="info">
  This is the same dashboard the CLI serves with `sfdt ui` and that the Chrome extension's bridge
  talks to. See [CLI → Web Dashboard](/cli/dashboard) for a full tour of its pages.
</Callout>

<Callout type="warning">
  The dashboard runs a local server bound to `127.0.0.1`. If its port is already taken, the
  extension recovers automatically when the occupant is a leftover sfdt dashboard from a previous
  session — it stops that stale server and retries. If the port is held by an **unrelated**
  process, the extension never kills it; instead it shows an error with an **Open Settings**
  action so you can point [`sfdt.dashboardPort`](/vscode-extension/settings) at a free port.
</Callout>
