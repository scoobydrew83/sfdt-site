# AI & Agents

sfdt's AI features are optional and gated behind `features.ai`. The provider is
selected by `ai.provider`:

| Provider | Requirement | Tools? |
|---|---|---|
| `claude` | Claude Code CLI | yes (agentic, read-only sandbox) |
| `gemini` | Gemini CLI | yes |
| `openai` | Codex CLI | yes |
| `http` | OpenAI-compatible endpoint (`ai.baseURL`, `ai.model`, `ai.apiKeyEnv`) | no (text only) |

The three CLI providers run as local agentic subprocesses; `http` is plain
text-completion and cannot run tools, so commands pre-gather context for it.
With `http`, prompt content is sent to the configured endpoint and is passed
through `redactSensitiveData` first.

## AI-powered commands

| Command | What AI does |
|---|---|
| `changelog generate` | Turn recent commits into CHANGELOG entries |
| `release` (notes) | Generate admin-facing release notes |
| `test` (on failure) | Diagnose Apex test failures |
| `explain` | Diagnose a deployment error log |
| `review` | Review the current diff |
| `quality --fix-plan` | Prioritized fix plan from quality output |
| `manifest --ai-cleanup` | Audit a draft package.xml for missing dependencies |
| `deploy --smart --ai-fix` | Explain a failed deploy, or run the auto-fix loop |
| `docs generate --roles` | Per-component, per-role documentation guides |
| `monitor/audit --notify` (with `notifications.summary.enabled`) | Executive-summary digest |

## Non-interactive `--agent` mode

`deploy`, `explain`, `review`, and `quality` accept `--agent`, which runs the AI
session non-interactively (it never blocks on input) — so coding agents and CI can
drive these commands. Combine with `--json` for machine-readable output where
supported.

## Bounded coding-agent auto-fix loop

`deploy --smart --ai-fix` can run a write-capable auto-fix loop when explicitly
enabled. It is off by default and layered with safeguards:

```jsonc
{ "ai": { "agent": { "enabled": false, "maxTurns": 3, "allowWrite": false } } }
```

- Requires `ai.agent.enabled` **and** `ai.agent.allowWrite`.
- CLI providers only (the `http` provider can't run tools).
- Bounded by `maxTurns` (override with `--max-turns`).
- Re-validates via a dry-run each turn and **never deploys**.
- The (attacker-influenceable) failure output is redacted before prompting.

## Editable prompts

All AI behavior is driven by editable prompt templates (override per project under
`.sfdt/prompts.json`). New templates:

- `deploy-error` — deploy/validation error resolution.
- `doc-apex`, `doc-flow`, `doc-lwc`, `doc-object` — per-metadata-type
  documentation guides (fall back to `doc-role-guide`).
- `monitor-summary` — org-health executive summary for notification digests.
