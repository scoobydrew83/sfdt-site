# Org Health ΓÇö Audit & Monitor

`sfdt audit` and `sfdt monitor` run native, read-only health checks against a
Salesforce org and write a normalized JSON snapshot consumed by the GUI, the
VS Code extension, and the MCP server.

Every check returns the same shape:

```json
{ "id": "limits", "title": "Org limits", "status": "ok | warn | fail | error", "summary": "ΓÇª", "findings": [ ΓÇª ] }
```

Snapshots are written to `logs/audit-latest.json` and `logs/monitor-latest.json`.
`audit all` / `monitor all` exit non-zero when any check is `fail` or `error`, so
they double as CI gates.

## Commands

```bash
sfdt audit all                 # run every audit check, write a snapshot
sfdt audit <check>             # run a single check (subcommands below)
sfdt audit all --json          # machine-readable output (no spinners)
sfdt audit all --notify        # also push the snapshot to notification channels
sfdt audit all --org myOrg     # target a specific org alias

sfdt monitor all               # run every monitoring check
sfdt monitor all --backup      # also take a full metadata backup
sfdt monitor backup            # backup only
sfdt monitor <check>
sfdt monitor all --notify --json
```

## Audit checks

| Check | What it flags |
|---|---|
| `audittrail` | Suspicious setup activity in the Setup Audit Trail |
| `licenses` | License types at/above the usage threshold |
| `mfa` | Active standard users with no registered MFA method |
| `unused-apex` | Apex classes with zero test coverage (heuristic) |
| `apex-unreferenced` | Apex classes with no inbound metadata dependency |
| `inactive-users` | Active users with no recent login |
| `api-versions` | Apex/triggers below the minimum API version |
| `inactive-flows` | Flow definitions with no active version |
| `inactive-validations` | Validation rules that are not active |
| `inactive-workflows` | Workflow rules that are not active (legacy automation) |
| `unused-permsets` | Permission sets with no user or group assignment |
| `connected-apps` | Connected apps that permit all users (admin approval off) |
| `field-descriptions` | Custom fields missing a description |
| `lint-access` | Custom objects with no Read access granted |
| `lint-access-fields` | Custom fields with no Read access granted (field-level) |

## Monitor checks

| Check | What it flags |
|---|---|
| `limits` | Org limits at/above the warning threshold |
| `errors` | Recent failed async Apex jobs |
| `health` | Security Health Check score below the floor |
| `org-info` | Instance, edition, and trial/expiration window |
| `deploy-history` | Recent deployment success/failure (latest failed ΓåÆ fail) |
| `deprecated-api` | Traffic on deprecated API versions (ApiTotalUsage logs) |
| `flow-errors` | Paused (potentially stuck) flow interviews |
| `backup` | Full metadata backup (with `--backup`) |

### Graceful degradation

Some checks depend on Beta or license-gated objects
(`MetadataComponentDependency`, `DeployRequest`, `FlowInterview`,
`ConnectedApplication`, `ValidationRule`/`WorkflowRule`, `EventLogFile`). When an
org can't run one of these, the check returns **`warn`** ("unavailable ΓÇª") rather
than `error`, so `audit all` / `monitor all` never fail CI just because an org
lacks a particular API or license.

## Surfaces

- **GUI**: the Org Audit and Org Monitor pages render the snapshot.
- **VS Code**: the Org Health tree shows each check; run any check from the
  Commands tree.
- **MCP**: `sfdt_audit` / `sfdt_monitor` (the `check` argument accepts any id
  above or `all`).
- **PR decoration**: `sfdt pr comment --type audit|monitor` posts the snapshot to
  a pull request (see Retrofit & PR comments).
